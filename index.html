Must Fix

4 issues
Severity Module Location Issue Suggested Fix
Security Token Service token.service.ts:321 decryptUniqueId silently falls back to empty-string key when JWT secret is missing. encryptUniqueId correctly throws on a missing secret, but decryptUniqueId uses ?? '', allowing decryption with a predictable empty key. If the secret is ever unset, an attacker who can craft a dispatch token could exploit this. Mirror encryptUniqueId: check for a missing secret and throw InternalServerErrorException before proceeding — same as the encrypt path.
Must Fix Dispatch dispatch.service.ts:492 URL injection — uniqueId not encoded. The parameter is directly interpolated into a URL string. If uniqueId contains &, #, =, or % (e.g. an email or corporate ID), it can break or alter the query string sent to rqillp. Use URLSearchParams: const url = new URL(`.../users`); url.searchParams.set('unique_id', uniqueid);
Must Fix External Auth external-auth.service.ts:126 User tokens cached with wrong TTL. getOrCreateUserToken caches the token using the default Valkey TTL instead of the actual expires_at from the rqillp API. If the token expires in 15 minutes but the default TTL is 1 hour, the cache will serve a stale expired token for 45 minutes. Derive TTL from expiresAt:
const ttl = expiresAt ? expiresAt - Math.floor(Date.now()/1000) : undefined;
await this.cacheService.set(userKey, token, ttl);
Must Fix Auth auth.service.ts:364–379 Duplicate JSDoc block — the same comment for exchangeExternalServiceToken appears twice in a row, left over from a previous edit. Remove one of the two identical JSDoc blocks.

Should Fix

6 issues
Severity Module Location Issue Suggested Fix
Performance LIS Completion lis-completion.service.ts:44–64 getECardFromRQILLPService is fetched unconditionally in processCertificateClaim even when eCard is disabled. The Promise.all always fires 4 API calls, then immediately exits early at line 70 when isEcardEnabled === false — wasting one round trip every time. Fetch the first 3 items in parallel, check isEcardEnabled, then conditionally fetch eCard master data only if needed.
Performance Auth Service auth.service.ts:181–187, 337–344 storeRefreshTokenJti and getOrgLanguage run sequentially after generateAuthTokens (synchronous), but are fully independent of each other. Affects both createRegisteredUserResponse and refreshAccessToken. await Promise.all([storeRefreshTokenJti(...), getOrgLanguage(...)])
Performance Registration registration.controller.ts:218–252 getCoursesByCourseIdWithCache and getOrgLanguage are called sequentially after authToken is obtained, but both are independent and only need authToken. Run both in a single Promise.all once authToken is available.
Code Quality Registration registration.service.ts:87–95 Pointless try/catch in registerLearner that only re-throws. Adds dead code with zero value — the error propagates identically without it. Remove the try/catch entirely.
Code Quality Course Cache course-list-cache.util.ts:34 getCoursesByCourseIdWithCache re-uses app.cache.orgValidationTtl as its TTL, tying course data staleness to org validation config. They have different semantics and should be independently configurable. Add a dedicated app.cache.courseListTtl config key with its own default.
Code Quality System Settings system-settings.util.ts:59 systemKey is interpolated directly into a URL query string without encoding — inconsistent with the rest of the codebase where URLSearchParams or new URL() is used. Use URLSearchParams to append filter[system_key].

Minor Dispatch dispatch.service.ts:583 let launchToken: string = '' initialised to an empty-string sentinel. If branching logic ever changes, the silent empty value could be used. Prefer let launchToken: string and let TypeScript enforce that both branches assign before use.
Minor Dispatch dispatch.service.ts:709–784 vendorAuthResponse['field'] = value uses bracket notation to bypass TypeScript's type checker. The response object should be built as a properly-typed object and assigned at the end.
Minor Curriculum curriculum.service.ts:281–306 Three of the four Promise.all HTTP calls omit Accept: 'application/json' headers that every other outbound call in the codebase includes. Inconsistent.
Minor Auth auth.service.ts:149, 189 meta: any used for the response shape in both createUnregisteredUserResponse and createRegisteredUserResponse. The shape is known — typed interfaces would surface API drift at compile time.
Minor Token Service token.service.ts:326 The createDecipheriv cast for GCM's setAuthTag is a Node.js typing gap. Extracting an aesGcmDecrypt helper isolates the cast and keeps decryptUniqueId readable.
