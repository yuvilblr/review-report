Must Fix

7
SeverityModuleLocationIssueSuggested Fix
Must Fix Dispatch dispatch.service.ts:85-222 validateProduct fetches ALL products for an org, then loops through the entire array in memory via Array.find() (line 125) to match a single vdrSubscriptionId. This is inefficient — wastes bandwidth, memory, and CPU for orgs with many products. Additionally, only the first page of results is checked, so if the target product falls on page 2+, validation incorrectly returns false, blocking legitimate users. Avoid the loop entirely by filtering server-side: GET /organizations/{orgId}/products?filter[id]={vdrSubscriptionId}. Returns only the matching product — no pagination issue, no in-memory scan, single result.
Must Fix Dispatch vendor-auth-request.dto.ts:43-44 userName has @IsString() but no @IsNotEmpty(). Empty string "" passes validation, causing split(',') to produce empty name fields in registration tokens. Add @IsNotEmpty() decorator to userName.
Must Fix Common user-license.repository.ts:65-82 getOrCreateUserLicense has a TOCTOU race condition. Concurrent requests for the same (user_id, org_id, course_id) both find no license, both create — Prisma P2002 duplicate key error bubbles up as 500. Catch P2002 errors and retry findFirst to return the existing record created by the other request.
Must Fix External Auth external-auth.service.ts:173-176 Gateway auth token cacheTtl = configuredTtl - bufferTime can go negative when configuredTtl < 300. redis.setex with negative TTL throws. Same issue for content service auth (line 253, bufferTime=60). const cacheTtl = Math.max(configuredTtl - bufferTime, 1);
Must Fix Axios axios.service.ts:58-63 httpCall destructures { method, url, data, ...config } but callers inconsistently pass data inside config or as the 4th arg. Config spread can override explicit data parameter. curriculum.service.ts:378-382 passes data inside config. Strip data from config before spreading: const { data: _ignored, ...restConfig } = config ?? {}; then spread restConfig.
Must Fix External Auth external-auth.service.ts:85 getOrCreateAuthToken caches with default Redis TTL, not the token's actual expiry (expires_at). If token expires sooner than the default TTL, requests will fail with stale tokens. Gateway and content service methods correctly apply buffer times, but the primary auth token does not. Parse expires_at from response and compute cacheTtl = expiresAt - now - bufferTime, similar to gateway/content auth methods.
Must Fix Common user-license.repository.ts:101-139 getAndExpireLicense has a race between findFirst and update. Another process could change the license status between check and update, and the update would overwrite that change. Add the status condition to the update's where clause: where: { id: licenseId, status: 'activated' }.

Security Findings

8
SeverityModuleLocationIssueSuggested Fix
Security Dispatch dispatch.service.ts:257 Vendor secret key included as plaintext in Redis cache key vendor:${vendorId}:${vendorKey}. Visible in Redis key scans, slow-query logs, and monitoring tools. Hash the vendorKey before using in cache key: createHash('sha256').update(vendorKey).digest('hex').slice(0,16).
Security Auth jwt.strategy.ts:19 Hardcoded fallback JWT secret 'your-secret-key'. In a misconfigured deployment, tokens signed with this public string would be accepted. Throw an error if the secret is missing: if (!secret) throw new Error('auth.jwt.secret is required').
Security Dispatch dispatch.service.ts:492 uniqueid interpolated directly into URL query string without encodeURIComponent(). Characters like &, #, or spaces cause URL malformation and potential query parameter injection. Use encodeURIComponent(uniqueid) or build URL with URL + searchParams.append().
Security Auth token.service.ts:322 decryptUniqueId falls back to empty string as key when auth.jwt.secret is missing. Uses a predictable, well-known key. Contrasts with encryptUniqueId (line 302) which correctly throws. Mirror encryptUniqueId behavior: throw InternalServerErrorException if secret is missing.
Security Auth token.service.ts:436-478 CacheService silent failure undermines launch token single-use guarantee. When Valkey is down, exists() returns false and set() silently fails — launch tokens can be replayed indefinitely. Treat cache failures as errors for security-critical operations. At minimum, log a warning when the used-check may be unreliable.
Security License license.controller.ts:31-44 GET /licenses takes filter[userId] as query param without verifying the authenticated user matches. Any authenticated user can enumerate any other user's licenses. Verify query.filter.userId === req.user.userId, or extract userId from JWT instead of query params.
Security Curriculum curriculum.service.ts:482-486 launchCourse returns oauth_consumer_key, oauth_nonce, and other OAuth fields in the response body. getCourseList correctly strips these via removeOAuthFields(), but launchCourse does not. Verify this is intentional for LTI form POST. If not needed by client, apply removeOAuthFields() consistently.
Security Dispatch dispatch.controller.ts:108-115 getCourseStatus logs full JSON-serialized response via JSON.stringify(response). Can inadvertently capture sensitive data as the response shape grows. Log only non-sensitive identifiers using structured logging with explicit fields.

Should Fix

15
SeverityModuleLocationIssueSuggested Fix
Performance Auth auth.service.ts:137-148 storeRefreshTokenJti, getTokens, and getOrgLanguage are awaited sequentially but are independent. Same pattern in refreshAccessToken and registration.controller.ts:247-252. Use Promise.all() for independent async operations.
Performance External Auth external-auth.service.ts:55-96 Classic thundering herd: concurrent requests all see "no cached token", all fetch from external service simultaneously. Applies to all getOrCreate* methods. Implement a local mutex/lock per cache key (in-memory Map of Promises) so only one request fetches while others await.
Code Quality Auth auth.service.ts:100-130 createUnregisteredUserResponse and createRegisteredUserResponse accept vendorDetails: any and build meta: any, bypassing all type checking. Use LaunchTokenPayload interface from token.service.ts instead of any.
Code Quality Dispatch vendor-auth-response.dto.ts DTO is missing fields assigned in service: planId, paymentMode, registerUrl, reccurenceCycle (typo: should be recurrenceCycle), suspendData. Bracket notation assignment bypasses type checking. Add missing fields to DTO and fix the typo.
Code Quality Auth + Registration auth.service.ts + registration.controller.ts:96-113 Duplicate extractBearerToken logic in both modules. Auth version validates JWT structure (3 parts), registration version does not and throws different exception types. Extract a shared utility function or reuse the auth service's method.
Code Quality Registration registration.controller.ts Controller has 7 direct dependencies and contains significant business logic (license creation, token generation, cookie handling). Violates thin-controller pattern. Move registration business logic (lines 180-282) into RegistrationService.
Code Quality Dispatch dispatch-course-status.dto.ts Dead DTO: replaced by CourseStatusRequestDto. Also has type inconsistency: usrSubscriptionId declared as string but decorated with @IsNumber() and parseInt transform. Remove the dead DTO and its export from the barrel file.
Correctness Cache cache.service.ts:33-35 get() uses naive JSON detection (startsWith('{')). A plain string starting with { is incorrectly parsed. Failed parse silently returns null, hiding data corruption. Always attempt JSON.parse(), fall back to raw string in catch block.
Correctness Common http-error.handler.ts:23 handleHttpError maps HTTP 403 (Forbidden) to UnauthorizedException (401). Different semantics: 403 means valid credentials but insufficient permission. Map 403 to ForbiddenException instead of UnauthorizedException.
Correctness Common system-settings.util.ts:86-89 getSystemSetting silently swallows ALL errors including auth failures and programming bugs. Root cause of failures hidden from callers. Re-throw non-transient errors (auth failures, 4xx). Only catch and return null for network/cache failures.
Performance License lis-completion.service.ts:188-209 triggerPostCompletionActions is awaited in processLisCompletion. HTTP response is blocked until certificate claim (multiple external API calls) completes. Can cause timeouts. Fire as fire-and-forget: this.triggerPostCompletionActions(...).catch(err => this.logger.error(...)).
Correctness Curriculum curriculum.service.ts:156-165 getCourseList continues with userLicense?.version when getAndExpireLicense returns null. External API calls proceed even without a license. Unclear if intentional. If license is required, throw NotFoundException when null. If optional, document this behavior.
Correctness Common lti-oauth.util.ts:65-67 buildCommonLtiFields creates lis_person_name_full with trailing/leading spaces when only first or last name is present. No .trim() applied. [first_name, last_name].filter(Boolean).join(' ')
Correctness Dispatch dispatch.service.ts:398-454 getOrgLanguage: empty API response fallback is cached, but error fallback is not. Error-case requests keep hitting the external service on every call. Either cache both fallbacks or neither — be consistent.
Code Quality Certificates certificates.service.ts:449 + lis-completion.service.ts:83 Duplicate fullName construction logic: full_name?.trim() || [first, middle, last].filter(Boolean).join(' ') in two places. Extract to a shared utility function: buildFullName(userDetails).
