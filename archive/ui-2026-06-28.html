Security Findings

4 issues
SeverityModuleLocationIssueSuggested Fix
Moderate courses course-info.tsx:35,98,105 Hand-rolled sanitizeHtml feeds two dangerouslySetInnerHTML sinks (course longDesc/launchDesc); it's bypassable and DOMPurify isn't installed. → High if course content is authored by untrusted/external parties; Moderate if only trusted admins author it. Add DOMPurify and replace sanitizeHtml with DOMPurify.sanitize().
Moderate external-api-client external-api-client.ts:18,29-39 External-service refresh tokens are held in the in-memory store and POSTed from the browser — XSS-exfiltratable, and it violates the project's HTTP-only-cookie convention for the external tier. Keep external refresh tokens out of JS; refresh via the LP backend exchange (HTTP-only cookie).
Moderate datadog datadog/index.ts:17-29 No beforeSend scrubbing while tracking resources/views — the auth ?token= param and the socket userId param are captured in RUM and sent to a third party (mask-user-input doesn't scrub URLs). Add a beforeSend that strips token/userId from resource.url and view.url.
Low auth routes/auth/exchange-token.tsx:28-29 The temp auth token arrives as ?token= and isn't stripped on the error path — leaks via history/referer. Short-lived/single-use token, so Low. Redirect with replace on success and to a clean path on failure; keep the token short-TTL/single-use.

Should Fix

16 issues
SeverityModuleLocationIssueSuggested Fix
Correctness course-socket course-socket.ts:27-32,79-82 On connect(newLicenseId), openSocket() early-returns if a socket is open, so the live socket stays on the old userId — completion events for the new course are missed. → Must Fix if users switch course/license in-session without a page reload. If licenseId changed and a socket exists, disconnect() before reopening.
Lifecycle course-socket course-socket.ts:27-35 connect() is called from useEffect with no cleanup, so the singleton socket + keep-alive interval outlive the view. Bounded (one socket; timers can accumulate only across repeated reconnects), not unbounded growth. Return () => disconnect() with ref-counting so co-owners don't tear down each other's socket.
UX routing routes/courses/index.tsx:16-29 No RequireAuth guard: on a hard refresh the in-memory store is empty, so a registered user sees a 401 fallback instead of being routed back through /auth. (Often acceptable if launch sessions aren't meant to survive refresh.) Add an auth guard that redirects to the launch/auth flow instead of inline-rendering the fallback.
Security lti / socket courses-view.tsx:46, course-socket.ts:56 Completion event uses postMessage('cdp_course_completed', '*') — wildcard target origin. Low impact (the payload is a fixed non-sensitive string), but it's a best-practice fix. Post to a specific configured parent origin.
Security lti lti-iframe.tsx:36 The LTI iframe grants allow="usb; serial; local-network-access; microphone *" and has no sandbox. Permission hardening. Scope allow to the content-service origin and only required features.
Correctness api-client api-client.ts:49-53 Proactive refresh is fire-and-forget and still returns the about-to-expire token, so a request in the buffer window can go out with a token that expires mid-flight (mitigated by the 401 retry). await the refresh when expiring-soon and return the new token, or rely solely on the 401 retry.
Resilience course-socket course-socket.ts:68 setInterval(openSocket, RECONNECT_MS) reconnect with no backoff and no retry cap. Self-rescheduling setTimeout with exponential backoff + ceiling.
Correctness auth-store use-auth-store.ts:87-97 getAccessToken() mutates state (set() to clear when expired) during a read — invoked in render it can trigger a mid-render update and extra re-renders. Move expiry-clearing out of the read path.
Correctness auth-store use-auth-store.ts:77-85 clearAuthData() doesn't reset source, leaving a stale proxy source after logout. Reset source: null.
Correctness api-client api-client.ts:32-37 Refresh response isn't validated; a 200 with an unexpected body calls setAccessToken(undefined, undefined) and poisons the store. Validate the refresh response shape first.
Resilience external-api-client external-api-client.ts:30-39 Tier-2 refresh has no try/catch around the fetch/res.json(), so a network error escapes instead of falling through to the tier-3 backend exchange. Wrap in try/catch so failures fall through.
Correctness courses courses-view.tsx:44-62 The all-completed effect reads t/toast/toastStyle but its dep array is only [...allCompleted] — stale-closure risk. Include the referenced values (or memoize).
Privacy error-fallback error-fallback.tsx:127 console.error logs the full error object (incl. stack/data) on every fallback render, in production. Gate behind isDevelopment.
Quality app shell main.tsx:14-16 <StrictMode> is commented out, so effect-cleanup double-invocation checks are lost (relevant to the socket/subscribe issues). Re-enable and fix anything it surfaces.
UX layouts layout-wrapper.tsx:22-33 Route Suspense and the loading branch both render a full-page loader (with a nested same-fallback Suspense), so every navigation flashes an empty page. Keep the prior Outlet mounted during transitions (overlay spinner).
i18n app / layouts not-found.tsx, footers/footer.tsx Hard-coded English strings (404 page, footer links/labels) with no i18n keys; footer links point to href="#". Use useTranslation keys; give footer links real hrefs.
